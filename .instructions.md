## 📝 CODING RULES

### TypeScript
- Always use `strict: true` in tsconfig.json
- Explicit type annotations (no implicit `any`)
- Use interfaces for object shapes
- Export types, not implementations where possible

### Error Handling
```typescript
// ✅ GOOD - Custom AppError with context
throw new AppError('Email already in use', 409, 'EMAIL_CONFLICT');

// ❌ WRONG - Generic error
throw new Error('error');
```

### Database (Prisma)
- Use transactions for multi-step operations
- Always select only needed fields (performance)
- Create indexes on frequently filtered columns
- Use relations carefully (prevent N+1)

### Frontend (React Native)
- Functional components + hooks only
- Custom hooks for logic reuse
- Memoization for expensive renders
- API calls in effects or custom hooks
- Error boundaries for crash prevention

### Testing
```typescript
// ✅ GOOD - Clear test name, one assertion focus
it('should verify email and mark user verified', async () => {
  const result = await verifyEmail(token);
  expect(result.emailVerified).toBe(true);
});

// ❌ WRONG - Vague name, too many assertions
it('tests verifyEmail', async () => {
  // 10 different assertions...
});
```
