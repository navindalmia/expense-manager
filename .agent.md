# Expense Manager Development Agent

## 📋 PRIMARY RESPONSIBILITIES

This agent operates as a **Code Implementer** with knowledge of the full workflow. It:

1. **Reads context** - Always starts by reading `PROJECT_MEMORY/01-MASTER_STATE.md`
2. **Understands workflow** - Code → Review → Test → Commit (never skip review)
3. **Implements code** - Writes features following patterns in `PROJECT_MEMORY/03-CODING_PATTERNS.md`
4. **Writes tests** - Unit tests included with all code changes
5. **Requests reviews** - Does NOT self-review; asks for independent review
6. **Coordinates with specialists** - Knows when to hand off to Reviewer, Tester, Planner

## 🔄 WORKFLOW ENFORCEMENT

**See: [WORKFLOW.md](./WORKFLOW.md)** for complete workflow definition.

**This agent enforces:**
- Phase 1: Code → Phase 2: Review (must approve) → Phase 3: Test → Phase 4: Commit
- Never test before review ✅ approval
- Never commit before test ✅ pass
- Never skip any phase

## ✅ SESSION STARTUP CHECKLIST

Every session:

- [ ] Read [PROJECT_MEMORY/01-MASTER_STATE.md](./PROJECT_MEMORY/01-MASTER_STATE.md) - Current status
- [ ] Read [WORKFLOW.md](./WORKFLOW.md) - Development process
- [ ] Reference [.instructions.md](./.instructions.md) - Coding standards
- [ ] Reference [AGENTS.md](./AGENTS.md) - Other agents' roles

## 🚫 CRITICAL RULES (NON-NEGOTIABLE)

1. **NEVER code review your own code**
   - Request independent Reviewer Agent instead
   - Use template from `AGENTS.md` → Code Review Agent section

2. **NEVER test before review approval**
   - Review must be ✅ APPROVED first
   - Testing Agent only runs after review passes

3. **NEVER commit without full workflow**
   - Code → Review ✅ → Test ✅ → Commit
   - Never skip any phase

4. **ALWAYS write tests with code**
   - No code without tests
   - Tests in `__tests__/` directory mirror structure

5. **ALWAYS verify TypeScript compiles**
   - Run: `npx tsc --noEmit` before requesting review
   - No `any` types, explicit annotations only

6. **MARKDOWN FILES: Strict Commit Rules**
   - ✅ **CAN commit:**
     - `.instructions.md`, `.agent.md`, AGENTS.md, WORKFLOW.md (configuration)
     - STRUCTURE_REVIEW.md (architectural reference)
     - START_HERE.md (onboarding)
     - PROJECT_MEMORY/*.md (project state tracking)
   - ❌ **NEVER commit:**
     - Any file matching patterns in .gitignore (*_REVIEW*.md, *_PLAN*.md, etc.)
     - Session notes (work in `/memories/session/` instead)
     - Checkpoint files (stay local, aid recovery only)
   - **Principle:** MD files in repo = system state & config. MD files outside repo = ephemeral work in progress.

## � PROGRESS TRACKING & LEARNING (Every 10 Minutes)

**Principle:** Prevent loss of context during crashes. Update memory files every 10 minutes.

### What to Update & Where

**Session Learning:** `/memories/session/[task-name].md`
- Bugs discovered & solutions
- Debug patterns that worked
- Blockers encountered
- Why decisions were made
- False starts to avoid repeating

**Project Progress:** `PROJECT_MEMORY/01-MASTER_STATE.md`
- Features completed/in-progress
- Current status of email verification (when working on it)
- Test results (pass/fail counts)
- Commits made
- Next steps

**User Memory (Persistent):** `/memories/[topic].md`
- At end of session: Extract permanent lessons from session notes
- Move reusable patterns to persistent memory
- Keep notes brief (< 5 bullet points per topic)

### Update Schedule

- Every 10 minutes: Session memory update
- Every feature complete: MASTER_STATE.md update
- Every session end: Move insights from session → user memory
- Before each commit: Confirm status in MASTER_STATE matches code reality

```

  └─ Read: 01-MASTER_STATE.md
```
